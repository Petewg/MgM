Il programma gestisce le prenotazioni di 5 risorse
controlla che le prenotazioni siano separate da un
intervallo minimo impostabile cliccando sulla statusbar.
Registra le cancellazioni e le evidenzia nel listato
ottenuto con control+alt+p sul secondo form, mentre se
eseguito nel primo form stampa le statistiche d'uso.
Nel giorno 8 aprile 2012 ho manualmente manomesso il dbf
per testare il comportamento grafico
Il programma se eseguito con il parametro PACK compatta
il database, se eseguito con il parametro ITA viene mostrato
con l'interfaccia in italiano,
se eseguito con il parametro NEW esegue una rielaborazione
del database: a partire dalla data scelta cancella i dati
precedenti e ne importa solo quelli in stato not deleted.
Se si cancella il database lo ricrea nuovamente.
Gli indici sono ricreati solo se usato da un solo utente.
Il file prenota.ini assegna il ritardo necessario fra
le prenotazioni ed eventualmente la lingua (default inglese).
Questa impostazione ed altri comandi sono disponibili come
opzioni usare presa /? per l'aiuto.

The program handles bookings of 5 resources
controls that bookings are separated by a
minimum interval set by clicking on the statusbar.
Record shows and cancellations in the listing
obtained with control + alt + p in the second form,
while if performed in the first form prints the usage statistics.
On the day April 8, 2012 I manually tampered with the DBF
to test the behavior graph
The program when run with the PACK compact the database,
if run with the ITA is shown with the interface in Italian.
When run with the NEW parameter performs a reworking
database: starting from the date range deletes the data
background and care only those being not deleted.
If you delete the database rebuilds it again.
The indexes are recreated only when used by one user.
The file prenota.ini assigns the necessary delay between
reservations and any language (default English).
This setting and other commands are available as
options to use presa /? for the help.
